# AI-Bro

Лендинг для внутреннего AI-движения компании — мотивирует сотрудников присоединиться к Telegram-боту, предлагать идеи и участвовать в AI-проектах.

## Tech Stack

- **Frontend:** React 18 + TypeScript + Vite
- **Styling:** Tailwind CSS + shadcn/ui (Radix UI primitives)
- **Animations:** Framer Motion
- **Routing:** React Router DOM v6
- **State/Data:** TanStack React Query
- **Linting:** ESLint 9
- **Testing:** Vitest + Testing Library
- **Scripts:** Python 3 (ask_ai.py — утилита для генерации идей через OpenAI API)

## Conventions

- **Коммиты:** на английском, императивный стиль ("Add feature", "Fix bug"), без префиксов (feat/fix)
- **Компоненты:** PascalCase, один компонент = один файл (.tsx)
- **Ветки:** формат `claude/<name>` для веток, созданных через Claude Code
- **UI-компоненты:** shadcn/ui — не модифицировать файлы в `components/ui/` напрямую, использовать как есть
- **Стили:** Tailwind utility classes, кастомные классы через `App.css` (gradient-text, hero-glow, etc.)
- **Язык интерфейса:** русский (контент лендинга на русском языке)

## What Claude Should Know

- Проект создан через **Lovable** (AI-конструктор) — папка с длинным UUID-именем это основной фронтенд
- Лендинг — одностраничник, все секции собраны в `pages/Index.tsx`
- CTA ведёт в Telegram (`https://t.me/...`) — ссылка пока не заполнена
- Анимации появления секций реализованы через `AnimatedSection` компонент (Framer Motion)
- `TelegramDialog` — диалоговое окно для перехода в Telegram-бот
- Python-скрипт `ask_ai.py` — вспомогательная утилита, не часть фронтенда

### Подсказки для эффективной работы

- **Все кастомные CSS-классы** (`gradient-text`, `gradient-accent`, `section-alt`, `hero-glow-*`) определены в `src/App.css` и `src/index.css` — перед созданием новых стилей сначала проверь, нет ли уже подходящего класса
- **`index.html` содержит OG-метатеги** с Lovable-preview изображением — при деплое на свой домен нужно обновить `og:image`, `twitter:image`, `og:title`, `og:description` и `<html lang="en">` → `<html lang="ru">`
- **Структура данных в секциях** (карточки в `WhySection`, шаги в `HowItWorksSection`, сообщения в `TelegramDialog`) хранятся в массивах-константах вверху каждого файла — для изменения контента редактируй массив, не JSX

## Don't

- **Не коммить `.env`** — файл уже содержит реальный API-ключ. Если ключ утёк, немедленно отзови его в dashboard OpenAI
- **Не модифицируй `components/ui/`** — это автогенерированные shadcn/ui компоненты, обновляются через CLI
- **Не переименовывай папку lovable-project** — она привязана к Lovable-проекту, переименование сломает связь
- **Не удаляй `hero-glow` CSS-классы** — они обеспечивают фоновые glow-эффекты на hero-секции
- **Не меняй язык контента на английский** — лендинг целевой для русскоязычной аудитории

### Дополнительные правила

1. **Не меняй порядок секций в `Index.tsx` без согласования**
   — *Что может пойти не так:* Лендинг — это воронка: Hero → Why → Benefits → How It Works → Topics → Telegram Dialog → CTA. Порядок выстроен по логике убеждения (зацепить → объяснить → показать → конвертировать). Перестановка ломает пользовательский путь.
   — *Пример:* Переместить `CtaSection` выше `WhySection` — пользователь увидит призыв к действию до того, как поймёт зачем ему это.

2. **Не трогай CSS-переменные цветов в `tailwind.config.ts` и `index.css`**
   — *Что может пойти не так:* Все цвета завязаны на HSL-переменные (`--primary`, `--background`, etc.). Изменение одной переменной каскадно сломает цвета по всему лендингу — кнопки, фоны, текст, карточки.
   — *Пример:* Поменять `--primary` с фиолетового на синий — сломается `gradient-accent`, `hero-glow`, иконки во всех `SectionCard`, аватарки в `TelegramDialog`.

3. **Не удаляй и не заменяй `AnimatedSection` wrapper из секций**
   — *Что может пойти не так:* `AnimatedSection` — это единый компонент для анимации появления всех секций (fade-up при скролле). Без него контент отрендерится, но пропадёт анимация появления, и лендинг будет выглядеть статично и дёшево.
   — *Пример:* Убрать `<AnimatedSection>` из `WhySection` — карточки появятся мгновенно без плавного входа, что будет резко контрастировать с анимированными соседними секциями.

4. **Не меняй шрифт Manrope на другой без согласования**
   — *Что может пойти не так:* Manrope задан как основной шрифт в `tailwind.config.ts`. Все размеры, межстрочные интервалы и отступы подобраны под его метрики. Замена шрифта сдвинет вёрстку — заголовки могут не влезть, кнопки поедут, карточки изменят высоту.
   — *Пример:* Заменить на Inter — он уже по ширине, заголовок в Hero перестанет переноситься в нужном месте, CTA-кнопка может стать двухстрочной на мобильных.

5. **Не хардкодь текст диалога бота в `TelegramDialog.tsx`**
   — *Что может пойти не так:* Сейчас сообщения хранятся в массиве `messages` вверху файла. Если начать вставлять текст прямо в JSX, станет невозможно быстро обновить диалог при смене сценария бота — придётся копаться в разметке вместо простого редактирования массива.
   — *Пример:* Заказчик просит поменять сценарий диалога. Если текст раскидан по JSX, легко пропустить одну реплику или сломать вёрстку при редактировании.
